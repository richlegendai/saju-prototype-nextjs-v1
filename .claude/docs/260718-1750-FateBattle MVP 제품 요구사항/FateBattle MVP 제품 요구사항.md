# FateBattle MVP 제품 요구사항

## 1. 제품 개요

FateBattle은 하나의 사주에서 밝은 면과 그림자 면을 꺼내 서로 논쟁시킨 뒤 현실적인 최종 판결을 내리는 한국어와 영어 모바일 우선 데모입니다. 장점을 따로 칭찬하고 약점을 따로 경고하는 운세가 아니라 같은 성향이 언제 힘이 되고 언제 발목을 잡는지 한 경기장에서 비교하는 것이 핵심 제품 명제입니다.

사주와 연간 흐름은 고정된 예시 콘텐츠로만 제공합니다. 실제 사주 계산, 상담, 결제, 로그인, 데이터 저장은 포함하지 않습니다.

- 제품명: FateBattle
- 한국어 태그라인: 좋은 운과 나쁜 운, 현실적인 최종 판결
- 영어 태그라인: Two Sides Enter. One Realistic Verdict.
- 기본 도메인: `fatebattle.com`
- 검색 정책: 프로토타입 전체 `noindex`, `nofollow`
- 데이터 정책: 출생 정보는 현재 브라우저 메모리 안에서만 사용

## 2. 목표

- 360px부터 430px까지 자연스럽게 동작하는 완성도 높은 모바일 프로토타입을 제공합니다.
- 한국어와 영어에서 같은 기능과 정보 구조를 제공합니다.
- 사주와 연간운세에서 밝은 면과 그림자 면의 논쟁부터 최종 판결까지 전체 흐름을 제공합니다.
- 긍정적인 말만 나열하는 운세와 구분되는 양면 대결 구조를 첫 화면에서 즉시 이해할 수 있게 합니다.
- 각 판결은 듣기 좋은 결론이 아니라 사용자가 바로 시험할 수 있는 현실적인 행동으로 끝냅니다.
- 개인정보가 URL, 저장소, 쿠키, API 요청에 남지 않는 데모 경계를 증명합니다.
- 나머지 서비스 카드와 하단 내비게이션도 실제 화면으로 연결하여 비활성 요소가 없도록 합니다.

## 3. 비목표

- 실제 사주 계산 또는 정확도 주장
- 실시간 AI 생성 또는 외부 AI API 호출
- 회원 가입, 실제 로그인, 결제, 구독
- 서버 API, 데이터베이스, 분석 도구, 사용자 추적
- 출생 정보 저장, 동기화, 공유

## 4. 기술 요구사항

- Next.js 16.2.10 App Router
- React와 TypeScript strict
- Tailwind CSS와 lucide-react
- Vitest, Testing Library, Playwright
- Server Component 우선, 상호작용 화면만 Client Component
- 루트 `proxy.ts`를 통한 브라우저 언어 자동 감지
- 모든 동적 `params`는 비동기로 처리

## 5. 로케일과 라우팅

지원 로케일은 `ko`, `en` 두 가지입니다.

- `/`는 `Accept-Language`가 한국어를 선호하면 `/ko`, 아니면 `/en`으로 이동합니다.
- 모든 제품 화면은 `app/[lang]` 아래에 둡니다.
- 언어 전환은 현재 경로를 유지합니다.
- 언어 쿠키에는 로케일 값만 저장합니다.
- 잘못된 로케일은 404로 처리합니다.
- 번역 사전은 정확한 공통 타입을 공유하여 누락 키가 타입 검사와 테스트에서 드러나야 합니다.

필수 경로:

- `/[lang]`
- `/[lang]/saju`
- `/[lang]/saju/result`
- `/[lang]/yearly`
- `/[lang]/yearly/result`
- `/[lang]/service/[serviceId]`

법적 및 데모 안내 화면은 서비스 화면 또는 푸터에서 접근할 수 있어야 합니다.

## 6. 디자인 시스템

선택한 방향은 Modern Coven입니다.

- Midnight Indigo: `#100B1C`
- Deep Plum: `#1C1330`
- Coven Purple: `#7C4D9E`
- Moon Gold: `#D8B96F`
- Bright Gold: `#E2C77F`
- Muted Crimson: `#A6556B`
- Moonlight Ivory: `#F5F1E8`

영어 제목은 정제된 serif 계열을 사용합니다. 한국어 제목은 Noto Serif KR 계열을 우선합니다. 본문은 Noto Sans KR 계열을 우선하되 외부 폰트 요청이 없어도 견고한 시스템 폴백으로 읽을 수 있어야 합니다.

이미지나 복제 자산 없이 CSS와 직접 만든 SVG 분위기 요소만 사용합니다. Bright는 달빛 금색, Shadow는 채도가 낮은 진홍과 플럼으로 구분하고 중앙에는 VS 또는 저울 모티프를 둡니다. 두 면은 선과 악을 뜻하지 않으며 종교적인 판정, 공포, 유혈 표현을 사용하지 않습니다.

기본 애니메이션은 사용하지 않습니다. 색상과 투명도 전환은 최대 150ms입니다.

## 7. 공통 앱 셸

- 데스크톱에서 가운데 정렬되는 최대 480px 앱 셸
- 직접 만든 달과 코븐 심볼, 제품명, 언어 전환, 데모 로그인 버튼을 포함한 헤더
- 콘텐츠 아래 법적 문서 링크와 데모 안내
- 고정 하단 내비게이션: Daily, Home, Add Chart, Coven Chat, Archive
- 모든 클릭 요소에 pointer 커서
- 비활성 상태는 not-allowed, 로딩 상태는 wait 커서
- 터치 영역 최소 44px

## 8. 홈

서비스 카드는 다음 순서를 유지합니다.

1. Four Pillars Battle
2. Compatibility Battle
3. 10-Year Cycle Battle
4. Yearly FateBattle
5. Timing Battle
6. Daily FateBattle
7. Challenge the Verdict

Four Pillars Battle과 Yearly FateBattle은 완전한 입력과 결과 흐름을 제공합니다. 나머지는 Bright 주장, Shadow 반론, 현실적인 판결로 이어질 미래 경기장의 작동 방식을 설명하는 데모 화면으로 이동합니다.

## 9. 사주 흐름

입력 항목:

- 선택 별칭
- 양력 또는 음력
- 필수 출생일
- 출생 시간과 시간 모름 전환
- 선택 국가와 도시
- 개인정보 비저장 동의

검증:

- 출생일 필수
- 올바르지 않은 날짜 형식 거부
- 미래 날짜 거부
- 시간 모름이면 시간 없이 제출 가능
- 시간 모름이 아니면 시간 필수
- 음력 윤달 입력을 제공할 경우 관련 입력 완결성 검증
- 동의 필수

결과:

- 별칭을 반영한 양면 테마
- Bright와 Shadow의 전체 주장
- 본성, 관계, 일과 돈, 타이밍의 4개 대결 라운드
- 각 라운드의 Bright 주장, Shadow 반론, 라운드 결론
- 장단점을 한 문장 안에서 직설적으로 종합한 최종 판결
- 판결을 현실에서 시험할 수 있는 구체적인 행동
- 개인정보 비저장 안내
- 객관적 계산이나 예측이 아닌 해석형 고정 데모라는 면책

결과는 고정된 양국어 fixture이며 점수, 확률, 확신도처럼 실제 계산으로 오해할 수 있는 수치를 사용하지 않습니다. Bright와 Shadow는 도덕적 선악이 아니라 같은 패턴의 유리한 작동과 불리한 작동을 뜻합니다.

## 10. 연간운세 흐름

입력 항목:

- 출생일
- 2026년부터 2030년까지 연도 선택, 기본값 2026년
- 개인정보 비저장 동의

결과:

- 선택한 연도의 Bright와 Shadow 전체 주장
- 관계, 커리어, 돈, 타이밍의 4개 대결
- 각 대결의 양쪽 주장과 결론
- 한 해의 현실적인 최종 판결과 행동
- 현지화된 12개월 선택 컨트롤
- 매달 Bright 우세, Shadow 우세 또는 팽팽함과 그 이유

월을 선택하면 같은 화면에서 우세와 이유가 즉시 바뀝니다. 점수, 확률, 확신도, 사건을 단정하는 표현을 사용하지 않습니다.

## 11. 개인정보와 보안 경계

출생일, 출생 시간, 국가, 도시는 다음 위치에 나타나면 안 됩니다.

- URL과 검색 파라미터
- localStorage와 sessionStorage
- 쿠키
- 애플리케이션 API 요청

서버 API 경로를 만들지 않습니다. 결과는 같은 클라이언트 화면에서 메모리 상태로 즉시 표시합니다. 언어 쿠키에는 `ko` 또는 `en`만 저장합니다.

## 12. 접근성

- 시맨틱 HTML과 논리적인 제목 순서
- 모든 입력에 연결된 label
- 키보드 탐색과 명확한 focus-visible 표시
- 일반 텍스트 명암비 최소 4.5:1
- 모바일 본문 최소 16px
- 터치 영역 최소 44px
- 가로 스크롤 금지
- 색상만으로 상태를 전달하지 않음
- `prefers-reduced-motion` 준수

## 13. 메타데이터와 법적 화면

- `https://fatebattle.com` 기준 canonical
- 한국어와 영어 hreflang
- robots 메타와 `robots.txt`에서 `noindex`, `nofollow`
- 개인정보 안내, 데모 이용조건, 오락 목적 면책 화면 접근 가능
- 실제 서비스, 의료, 법률, 재무 조언으로 오해하지 않도록 명확히 안내

## 14. 검증 기준

단위 테스트:

- 한국어와 영어 번역 키 완전 일치
- 두 언어 모두 사주 4라운드, 연간 4대결, 12개월 우세 fixture 제공
- 미래 날짜 거부
- 동의 누락 거부
- 시간 모름 제출 허용
- 일반 시간 입력 누락 거부

Playwright:

- 390x844 영어 사주 흐름에서 별칭 Mira, 생년월일 1992-08-14, 시간 모름, 동의를 입력하고 4개 대결 라운드, Bright와 Shadow, Battle Verdict, 현실적인 행동 확인
- 한국어 연간운세에서 출생일, 2028년, 동의를 입력하고 4개 대결, 올해의 최종 판결, 12개월, 2월 우세와 이유 확인
- 브라우저 로케일 리다이렉트
- 현재 경로 유지 언어 전환
- 서비스 카드 7개와 하단 내비게이션 5개 목적지
- URL, 저장소, 쿠키, 네트워크 개인정보 비노출
- 360x800, 390x844, 430x932 가로 오버플로 없음

품질 명령:

- `pnpm lint`
- `pnpm test`
- `pnpm exec playwright test`
- `pnpm build`

모든 명령은 경고나 오류 없이 통과해야 하며 검증 로그와 화면 캡처를 `.artifacts/verification/` 아래에 저장합니다.
