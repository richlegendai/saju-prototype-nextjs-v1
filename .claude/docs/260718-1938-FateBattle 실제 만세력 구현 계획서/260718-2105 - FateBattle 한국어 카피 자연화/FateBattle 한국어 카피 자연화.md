# FateBattle 한국어 카피 자연화 구현 계획서

**목표:** FateBattle 한국어 화면의 뜻과 기능 안내는 그대로 유지하면서 번역투와 기계적인 대결 표현을 자연스러운 서비스 문구로 다듬습니다.

**접근 방식:** `lib/locales/ko.ts`를 한국어 카피의 단일 수정 대상으로 삼습니다. 화면 구조, 번역 키, 영어 문구, 계산 로직은 변경하지 않습니다. 순수 카피 작업이므로 문구 존재를 고정하는 테스트를 새로 만들지 않고 의미 감사와 실제 모바일 화면 검증을 사용합니다.

**기술 스택:** Next.js 16, TypeScript, 정적 로케일 사전, Vitest, Playwright

## 보호할 의미

- `FateBattle` 제품명
- 빛의 마녀, 그림자 마녀, 운명 판관의 역할
- 외부 라이브러리로 계산한 사주 원국과 규칙 기반 해석의 구분
- 양력 1800년부터 2300년, 음력 1800년부터 2100년
- 연도 선택 범위 2026년부터 2030년
- 출생정보를 현재 탭의 메모리에서만 사용하는 개인정보 원칙
- 오락과 성찰용 데모이며 사건 예측이나 전문 조언이 아니라는 면책

## 작업 1: 한국어 카피 감사

**파일**

- 검토: `lib/locales/ko.ts`
- 대조: `lib/locales/en.ts`

1. 홈, 사주, 연도별 운세, 서비스 안내, 개인정보와 면책 문구를 구분합니다.
2. `맞붙입니다`, `경기장`, `대결`, `판결`이 지나치게 반복되는 문장을 찾습니다.
3. 번역투, 추상적인 명사형, 과한 비유, 모바일에서 길게 늘어지는 문장을 표시합니다.
4. 고유명사, 수치, 기능 제약, 개인정보 의미를 변경 금지 항목으로 고정합니다.

## 작업 2: 카피 자연화

**수정**

- `lib/locales/ko.ts`

1. 전체 재작성은 한 번만 수행합니다.
2. 브랜드의 대결 콘셉트가 필요한 제목과 핵심 버튼에는 `대결` 또는 `판결`을 남깁니다.
3. 설명 문장에서는 일상적인 동사와 짧은 문장으로 풀어 씁니다.
4. 원문에 없는 기능, 정확도, 감정, 수치, 성과는 추가하지 않습니다.

## 작업 3: 후처리 감사

**검토**

- `lib/locales/ko.ts`
- `lib/locales/en.ts`

1. 1차 감사에서 제품명, 캐릭터 역할, 계산 범위, 개인정보와 면책 의미를 대조합니다.
2. 2차 감사에서 번역투, 반복 어미, 기계적인 병렬, 같은 단어 반복을 점검합니다.
3. 변경률이 과도하면 이미 자연스러운 문장은 원문으로 되돌립니다.

## 작업 4: 자동 검증

다음 명령이 모두 종료 코드 0인지 확인합니다.

```bash
pnpm exec tsc --noEmit --pretty false
pnpm lint
pnpm test
pnpm exec playwright test
pnpm build
git diff --check
git diff --cached --check
```

## 작업 5: 모바일 화면 검증

1. QA 전용 3100 포트에서 프로덕션 서버를 실행합니다.
2. 390x844 화면으로 `/ko`와 `/ko/saju`를 엽니다.
3. 홈 핵심 문구와 사주 입력 문구가 자연스럽게 읽히는지 확인합니다.
4. 가로 넘침, 콘솔 오류, 기존 영어 문구 노출이 없는지 확인합니다.
5. 스크린샷을 `.artifacts/verification/`에 저장합니다.
6. 브라우저와 QA 서버를 종료하고 3100 포트가 비었는지 확인합니다.

## 완료 조건

- 사용자가 지적한 문장이 자연스럽게 바뀝니다.
- 인접한 한국어 카피의 번역투와 기계적인 반복도 함께 정리됩니다.
- 영어 사전과 번역 키 구조가 그대로 유지됩니다.
- 전체 자동 검증과 실제 모바일 화면 검증이 통과합니다.
- commit과 push는 실행하지 않습니다.

## 후속 수정: 사주 입력 화면 제목과 소개

사용자가 이전 사주 입력 화면의 `두 얼굴을 맞붙입니다`, `고정된 데모 해석`, `양쪽 가능성` 같은 표현도 어색하다고 지적했습니다. 현재 소스에서는 일부 문구가 이미 바뀌었지만 소개 문장에 설명체와 번역투가 남아 있어 다음처럼 다시 정리합니다.

- 제목: `내 사주, 좋은 점만 보지 않습니다`
- 소개: `강점이 언제 힘을 발휘하고 약점이 어떤 상황에서 반복되는지 함께 살펴봅니다. 입력한 정보는 전송하지 않으며 결과는 미리 준비된 데모 해석입니다.`

보호할 의미는 강점과 약점을 함께 보여준다는 점, 입력 정보를 전송하지 않는다는 점, 결과가 실제 AI 판정이 아닌 미리 준비된 데모라는 점입니다. `lib/locales/ko.ts`의 두 값만 수정하고 변경 전후의 `/ko/saju` 390x844 화면을 캡처합니다. 이후 전체 타입, 린트, 테스트, 빌드와 diff 검사를 다시 실행합니다.
