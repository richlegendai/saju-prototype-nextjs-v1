# FateBattle 실제 만세력 세 마녀 설계

## 확정 사항

- 만세력 계산은 외부 라이브러리를 사용합니다.
- Bright Witch는 가능성, 장점, 성장 조건을 주장합니다.
- Shadow Witch는 약점, 자기기만, 반복 위험을 반박합니다.
- Fate Judge는 마녀이며 양측의 주장을 종합합니다.
- 동일한 정규화 입력은 동일한 판결을 반환하고 현재 탭의 메모리 캐시에서 재사용합니다.
- AI API, 서버 저장소, 계정, 결제는 이번 데모에 추가하지 않습니다.

## 라이브러리 선택

### 선택: `manseryeok@2.0.0`

선택 이유:

- 한국천문연구원 기준 음력 데이터를 사용합니다.
- 연주는 입춘 절입 순간, 월주는 절기 경계를 기준으로 계산합니다.
- 양력과 음력 및 윤달을 지원합니다.
- TypeScript 타입을 제공합니다.
- 런타임 의존성이 없습니다.
- 한국 표준시와 진태양시 보정 옵션을 제공합니다.
- 사주 계산 범위는 양력 1800년부터 2300년, 음력 1800년부터 2100년입니다.

이번 데모는 입력된 시각을 그대로 사용하며 진태양시 보정은 적용하지 않습니다. 해외 출생지 시간대와 경도 보정은 도시 좌표 데이터가 필요한 후속 범위입니다.

### 검토한 대안

1. `@fullstackfamily/manseryeok`
   - 한국형 API와 비교적 단순한 사용법이 장점입니다.
   - 지원 범위와 절기 데이터 범위가 더 좁아 이번 선택에서 제외합니다.

2. `lunar-javascript`
   - 오래된 사용 사례와 많은 다운로드가 장점입니다.
   - 중국 표준시 중심의 경계값이 한국 기준 사주와 다를 수 있어 직접 사용하지 않습니다.

## 데이터 흐름

```text
사용자 입력
→ 클라이언트 검증
→ 입력 정규화
→ 메모리 캐시 조회
→ manseryeok 원국 계산
→ 오행 및 일간 요약
→ Bright Witch 주장
→ Shadow Witch 반론
→ Fate Judge 판결
→ 같은 탭의 메모리 캐시에 결과 보관
```

생년월일과 출생 시각은 URL, 쿠키, localStorage, sessionStorage 또는 애플리케이션 API에 기록하지 않습니다.

## 시간 미상 처리

- 라이브러리에는 날짜와 정오를 전달하여 연주, 월주, 일주를 계산합니다.
- 정오 값은 날짜 경계를 피하기 위한 내부 계산값일 뿐 시주로 노출하지 않습니다.
- 결과 화면의 시주는 `시간 미상` 또는 `Unknown`으로 표시합니다.
- 판결은 연주, 월주, 일주와 여섯 글자의 오행 분포를 사용합니다.

## 결정적 판결과 캐시

- 캐시 키는 버전, 달력 유형, 날짜, 시간 또는 시간 미상, 윤달 여부를 정규화하여 만듭니다.
- 별칭, 국가, 도시는 계산과 캐시 키에서 제외합니다.
- 동일 키는 같은 `SajuReading` 객체를 반환합니다.
- 판결 변형은 캐시 키와 계산된 일간 및 오행 분포로 결정합니다.
- 향후 AI를 연결할 때도 이 캐시 키를 서버 캐시의 idempotency key로 사용할 수 있습니다.

## 세 마녀 인터페이스

### Bright Witch

- 색상: 문 골드와 따뜻한 아이보리
- 아이콘: 태양과 마법 지팡이
- 역할: 계산된 일간과 우세 오행이 발휘하는 강점 설명

### Shadow Witch

- 색상: 로즈 플럼
- 아이콘: 초승달
- 역할: 같은 기질이 과해질 때 나타나는 비용과 반복 패턴 설명

### Fate Judge

- 색상: 코븐 퍼플과 문 골드
- 아이콘: 저울과 별
- 역할: 한쪽의 승리를 선언하지 않고 조건이 붙은 최종 판결과 행동 제시

모바일에서는 Bright와 Shadow를 두 칸으로 배치하고 Fate Judge는 전체 너비로 배치합니다. 360px 이하에서도 역할명과 설명이 잘리지 않아야 합니다.

## 실제 원국 카드

- 연주, 월주, 일주, 시주를 2열 카드로 표시합니다.
- 각 주는 한글 간지와 한자를 함께 표시합니다.
- 시간 미상은 시주 카드에서 명시합니다.
- 일간 오행과 전체 오행 분포를 함께 표시합니다.
- 계산값과 해석 문구를 시각적으로 분리하여 실제 계산과 엔터테인먼트 해석을 혼동하지 않게 합니다.

## 안전 및 한계

- 실제 계산값은 외부 라이브러리를 사용하지만 해석과 판결은 고정 규칙 기반 데모입니다.
- 건강, 법률, 재정, 사건 예측을 단정하지 않습니다.
- 도시 입력은 이번 데모에서 계산에 사용하지 않는다고 안내합니다.
- AI 호출과 영구 캐시는 후속 단계입니다.
