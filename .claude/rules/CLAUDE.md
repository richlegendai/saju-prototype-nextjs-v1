# 프로젝트 규칙

- 한국어 존댓말을 사용합니다.
- 승인된 범위만 수술적으로 변경합니다.
- TypeScript strict 모드를 유지하고 `any`와 타입 오류 억제를 사용하지 않습니다.
- 기능 변경은 실패 테스트, 최소 구현, 통과 테스트 순서로 진행합니다.
- 생년월일, 출생 시간, 국가, 도시는 브라우저 메모리 밖에 저장하거나 전송하지 않습니다.
- 모든 클릭 요소는 명확한 커서와 포커스 상태를 제공하며 터치 영역은 최소 44px입니다.
- 기본 애니메이션은 사용하지 않고 색상과 투명도 전환만 150ms 이하로 제한합니다.
- 생성 파일에 개인 식별 정보와 변경 이력 헤더를 넣지 않습니다.
- em dash 문자를 사용하지 않습니다.
- 명시적 승인 전에는 Git commit과 push를 실행하지 않습니다.
