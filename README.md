# FateBattle

> 좋은 점도 나쁜 점도, 있는 그대로.

FateBattle은 한국 사주를 좋게만 풀이하지 않는 모바일 우선 운세 서비스 프로토타입입니다. 같은 성향의 강점과 약점을 두 마녀가 각각 해석하고, 운명 판관이 두 관점을 종합해 현실적인 결론을 제시합니다.

현재 버전은 한국어와 영어를 지원합니다.

## 세 마녀

- **빛의 마녀 (Bright Witch)**: 가능성과 강점이 살아나는 조건을 찾습니다.
- **그림자 마녀 (Shadow Witch)**: 약점과 자기기만, 반복될 위험을 짚습니다.
- **운명 판관 (Fate Judge)**: 두 해석을 종합해 지금 적용할 수 있는 결론을 내립니다.

## 현재 구현 범위

| 기능 | 상태 | 설명 |
| --- | --- | --- |
| 사주 양면 대결 | 데모 제공 | 생년월일과 출생 시각으로 사주 원국을 계산하고 강점과 약점을 함께 풀이합니다. |
| 연도별 운세 | 데모 제공 | 2026년부터 2030년까지 연간 주제와 열두 달의 예시 해석을 제공합니다. |
| 궁합 | 미리보기 | 서비스 구성과 세 마녀의 역할만 확인할 수 있습니다. |
| 10년 대운 | 미리보기 | 장기 흐름을 다룰 화면 구성을 보여줍니다. |
| 택일 | 미리보기 | 행동할 때와 기다릴 때를 비교하는 구성을 보여줍니다. |
| 오늘의 운세 | 미리보기 | 하루의 강점과 주의점을 다룰 구성을 보여줍니다. |
| 결론 다시 살펴보기 | 미리보기 | 같은 질문을 두 관점에서 다시 살펴보는 흐름을 보여줍니다. |

### 사주 계산

- `manseryeok` 2.0.0으로 연주, 월주, 일주와 시주를 계산합니다.
- 양력과 음력, 음력 윤달, 출생 시각 미상 입력을 지원합니다.
- 오행 분포와 세 마녀의 해석을 한 화면에서 확인할 수 있습니다.
- 같은 입력으로 다시 계산하면 같은 결과가 나오며 계산 결과는 브라우저 메모리에만 잠시 보관합니다.
- 국가와 도시는 선택 항목입니다. 현재 프로토타입에서는 시간대나 진태양시 보정에 사용하지 않습니다.

## 지원 언어와 경로

루트 경로는 브라우저 언어 또는 저장된 언어 설정을 확인해 한국어와 영어 화면으로 이동합니다. 한국어가 아니거나 지원 언어를 확인할 수 없으면 영어를 사용합니다.

| 경로 | 설명 |
| --- | --- |
| `/` | 브라우저 언어에 따라 `/ko` 또는 `/en`으로 이동 |
| `/ko`, `/en` | 언어별 홈 |
| `/[lang]/saju` | 사주 입력과 결과 |
| `/[lang]/saju/result` | 직접 접근 안내 화면 |
| `/[lang]/yearly` | 연도별 운세 입력과 결과 |
| `/[lang]/yearly/result` | 직접 접근 안내 화면 |
| `/[lang]/service/[serviceId]` | 나머지 서비스와 법적 안내 미리보기 |

언어를 바꾸면 현재 경로를 유지하며 선택한 언어만 `saju-locale` 쿠키에 저장합니다.

## 개인정보 처리

출생 정보는 현재 탭의 브라우저 메모리에서만 처리합니다.

- 생년월일, 출생 시각, 국가와 도시는 서버로 전송하지 않습니다.
- 출생 정보를 URL, 쿠키, `localStorage`, `sessionStorage` 또는 데이터베이스에 저장하지 않습니다.
- 새로고침하거나 탭을 닫으면 입력 결과가 사라집니다.
- 언어 설정만 쿠키에 저장할 수 있습니다.
- 애플리케이션 API와 분석 도구를 사용하지 않습니다.

## 시작하기

### 요구 환경

- Node.js 20.9.0 이상
- pnpm 11.13.1

Next.js 16의 최소 Node.js 버전은 20.9.0입니다. 자세한 내용은 [Next.js 16 업그레이드 가이드](https://nextjs.org/docs/app/guides/upgrading/version-16)에서 확인할 수 있습니다.

### 설치

```bash
pnpm install
```

별도의 환경 변수나 외부 서비스 계정은 필요하지 않습니다.

### 개발 서버

```bash
pnpm dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

### 프로덕션 빌드

```bash
pnpm build
pnpm start
```

프로덕션 빌드는 기본 `.next` 폴더와 충돌하지 않도록 `.next-build`에 생성됩니다.

## 테스트

```bash
# TypeScript
pnpm exec tsc --noEmit --pretty false

# ESLint
pnpm lint

# 단위 테스트
pnpm test

# E2E 테스트
pnpm exec playwright test

# 프로덕션 빌드
pnpm build
```

Playwright 브라우저가 설치되어 있지 않다면 처음 한 번 다음 명령을 실행합니다.

```bash
pnpm exec playwright install chromium
```

테스트는 다음 항목을 확인합니다.

- 한국어와 영어 로케일 구조
- 입력값 검증과 음력 날짜 처리
- 실제 사주 원국과 오행 계산
- 같은 입력에 같은 결과가 나오는지 여부
- 세 마녀와 최종 판결 표시
- 연도와 월별 운세 흐름
- 출생 정보가 URL, 저장소, 쿠키와 애플리케이션 요청에 포함되지 않는지 여부
- 360px, 390px와 430px 모바일 화면의 가로 넘침
- 접근성 이름과 최소 터치 영역

## 프로젝트 구조

```text
app/
  [lang]/                 언어별 App Router 페이지
components/               폼, 결과, 세 마녀와 공통 UI
lib/
  locales/                한국어와 영어 사전
  saju-reading.ts         사주 원국 계산과 메모리 캐시
  validation.ts           출생 정보 검증
tests/
  unit/                   로케일, 입력 검증과 사주 계산 테스트
  e2e/                    모바일 흐름, 개인정보와 라우팅 테스트
proxy.ts                  루트 언어 감지와 리디렉션
```

## Vercel 배포

1. 저장소를 GitHub, GitLab 또는 Bitbucket에 올립니다.
2. Vercel에서 저장소를 가져옵니다.
3. Framework Preset은 Next.js를 사용합니다.
4. Install Command는 `pnpm install`, Build Command는 `pnpm build`를 사용합니다.
5. 환경 변수 없이 배포합니다.

`packageManager`가 `pnpm@11.13.1`로 고정되어 있어 Vercel이 pnpm을 감지할 수 있습니다. 커스텀 도메인은 배포 후 Vercel 프로젝트의 Domains 설정에서 연결합니다.
